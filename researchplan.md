Fast Yet Publishable Plan for Spectral Episode Temporal Mixer

The goal of this project is to keep the training profile close to a compact waveform CNN while making the research contribution more distinct than a front-end spectral gate. The proposed method, Spectral Episode Temporal Mixer (SETM), does that by moving spectral reasoning from signal manipulation to computation control. A tiny FFT-based summary does not rewrite the waveform. Instead, it chooses a spectral episode, and that episode determines which temporal experts are emphasized as the signal flows through a lightweight mixer backbone.

That framing is important because it creates a better research question. The old question was whether learned spectral filtering helps a fixed backbone. The new question is whether coarse spectral state can control temporal computation cheaply and usefully. That is more novel, easier to defend as an architectural contribution, and still compatible with a single-accelerator budget.

The architecture should remain intentionally compact. The waveform first goes through a real FFT summary stage that produces a small vector of band-energy features. A learned prototype library maps that summary to a soft mixture over spectral episodes. Each episode stores a routing template for a stack of temporal blocks. The temporal backbone itself is a sequence of residual mixer blocks built from depthwise temporal experts, pointwise channel mixing, and a small MLP. This is not an Inception variant and it is not a transformer. The computational heavy lifting remains 1D convolution, which is exactly why the model should stay fast.

The minimum causal ablation package is four-way. First, `uniform` routing tests the same backbone without adaptive control. Second, `static` routing tests whether a single learned schedule is enough. Third, `direct` routing tests whether the prototype library adds value beyond a plain MLP. Fourth, `prototype` routing is the full method. If the full method beats `uniform` and `static` but not `direct`, then the paper becomes a weaker story about adaptive routing in general rather than spectral episodes specifically. That is still potentially publishable, but it is a narrower contribution. If `prototype` beats all three at similar runtime, the main story is much stronger.

The core datasets should remain PTB-XL and CHB-MIT, with Sleep-EDF as the preferred third benchmark if time allows. PTB-XL tests whether the method survives multilabel imbalance and ECG morphology. CHB-MIT tests whether the method is useful in EEG seizure detection with subject-aware splits. Sleep-EDF is valuable because spectral state is clinically intuitive there, which makes the mechanism plots easier to interpret. The paper should not claim universal improvement. The bar is lower and more honest: improve Macro-F1 or balanced accuracy on at least one major track without severe regression elsewhere, while staying close to baseline training cost.

The main risk is not that the model will be too slow. The main risk is that the prototype library collapses into either uniform behavior or an unnecessarily complicated version of direct routing. That is why the first diagnostics should include prototype-usage histograms, route variability across blocks, and seed stability. If all samples use essentially one prototype, the contribution collapses. If the routes change but the metrics do not, the mechanism is active but not useful. If the runtime cost is much higher than expected, the backbone should be simplified before more experiments are run.

A realistic one-week plan is straightforward. Day 1 is implementation and smoke testing. Day 2 is PTB-XL `uniform` versus `prototype`. Day 3 is CHB-MIT `uniform` versus `prototype`. Day 4 runs `static` and `direct` ablations on the dataset where the signal looks strongest. Day 5 is a small validation-only sweep over number of bands, number of prototypes, and route-smoothness regularization. Day 6 is profiling, significance summaries, and mechanism plots. Day 7 is writing with explicit limitation framing. The point of this schedule is to detect whether the architecture has a real signal before spending time polishing a weak story.

The claim language should remain conservative. Good phrasing is that spectral episode routing is a promising fast inductive bias for biomedical time-series classification, that its benefits are dataset dependent, and that its main value is improving the computation-allocation story rather than rewriting the waveform. Bad phrasing would be that the learned prototypes are causal physiological states, that the method is universally superior, or that the mechanism is clinically validated. If the experiments work, SETM has a plausible path to a main-track submission because it is now a real architecture story rather than a small wrapper around someone else's backbone. If the experiments do not work, the diagnostics should make it obvious quickly, which is exactly what a good research plan is supposed to do.
